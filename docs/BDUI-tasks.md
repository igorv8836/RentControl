### BDUI-Client(клиентская библиотека для BDUI)
Задачи:
1) Реализовать sticky возможности(элементы, которые при скролле прикрепляются в началу или концу экрана), реализовать ui тесты со всеми тест-кейсами, а также обновить контракт на клиенте и на бэкенде
2) [x] Поменять подход с OverlayAction и PopupAction, они должны иметь такие же параметры как ForwardAction, кроме presentation, обновить контракт на бэкенде и клиенте, добавить ui тесты(и другие при необходимости). PopupAction должен уметь открываться снизу, посередине и сверху экрана(управление должно быть через контракт), он открывается поверх текущего экрана. Навигационные экшены должны быть устроены так, что сперва открывается remoteScreen как превью, если != null. Если есть path, то загружается основной экран. Если нет path, то remoteScreen становится основным экраном
3) Также нужно реализовать новый экшен RemoteAction на клиенте и бэкенде, по сути это как ForwardAction, но вместо верстки возвращает список экшенов. Кейс для использования: RemoteAction используется на клиенте в нестандартных сценариях, н-р, при входе в приложение будет RemoteAction на определенную ручку с параметрами, в ответ придет н-р ForwardAction на главный экран, если успешная авторизация. Также напиши тесты для этого
4) Изменить подход при построении экрана, экран должен строиться посекционно. Нужно вместе обсудить прежде чем делать!
