# TODO: BDUI Client (полный функционал поверх MVP)

Цель “full”: расширяемость, богатая верстка, производительность, безопасность и продвинутая диагностика.

## Layout / Styling система

- [ ] Общие модификаторы (padding/margin/size/weight/alignment/offset/zIndex) для всех `ComponentNode`.
- [ ] Border/shape/cornerRadius/shadow/elevation.
- [ ] Глобальная тема + токены (цвета/типографика/spacing) с возможностью переопределения на экране/секции/элементе.
- [ ] Анимации: появления/исчезновения, переходы между экранами, анимации состояний компонентов.

## Контент и медиа

- [ ] Реальный image loader (кеш, placeholder/error, prefetch).
- [ ] Rich text (links, spans, inline icons).
- [ ] Видео/анимации (опционально).

## Списки и сложная прокрутка

- [ ] Sticky headers “как в списке” (занимают место), помимо overlay sticky.
- [ ] Виртуализация/пагинация по секциям, “endless list” паттерны.
- [ ] Сохранение позиции скролла и состояния секций при навигации назад.

## Формы и валидации

- [ ] Нормальная модель формы (field state, validation rules, submit flows).
- [ ] Input types: numeric, phone, email, masked, multiline, ime actions.
- [ ] Серверные ошибки валидации: маппинг на поля и сообщения.

## Actions / Navigation / Effects

- [ ] RemoteAction (цепочки экшенов) + интеграционные тесты (клиент ↔ сервер).
- [ ] “Effects” слой: toast/snackbar/dialog/navigation как экшены.
- [ ] Расширяемая система кастомных экшенов без модификации ядра (регистрация + сериализация).

## Observability и качество

- [ ] Полная трассировка: network + render + actions + triggers (время, причины, контекст).
- [ ] Отправка логов/метрик на сервер (через отдельную реализацию логгера).
- [ ] Performance: измерения времени рендера, количество recompositions, оптимизации дерева.

## Безопасность и надежность

- [ ] Верификация payload (подпись/хеш), защита от “подмены экрана”.
- [ ] Политики контента: allowlist доменов для картинок/эндпоинтов.
- [ ] Строгая схема (JSON schema) + валидация на клиенте/сервере.

## Tooling

- [ ] Генерация документации из контракта (авто-список компонентов/полей).
- [ ] Визуальный preview (например, “playground” экран/десктоп тул).
- [ ] Snapshot/UI tests: golden images, семантические проверки.
- [ ] CI задачи: проверка совместимости контракта, линты, форматирование, миграции.
