# BDUI — Возможные улучшения

- **Адаптивные лэйауты**: серверные breakpoints (phone/tablet/desktop), responsive‑токены отступов, разные scaffold по ориентации.
- **Тёминг**: удалённые цветовые/типографические токены, динамические палитры (light/dark/high‑contrast), переопределения на экране.
- **Формы и валидация**: встроенные инпуты, sync/async валидаторы, маппинг ошибок, submit/remote actions с retry‑политиками.
- **Навигация DSL**: вложенные потоки, guarded routes (auth/feature flags), сохранение таб‑состояния, deep links и снимки back stack.
- **Доставка данных**: пресеты пагинации (cursor/page), streaming/long‑poll обновления, delta‑payload, оптимистичные апдейты.
- **Кеш и офлайн**: политики кеша экрана/контента, stale‑while‑revalidate, offline‑очередь действий, версионированные схемы.
- **Доступность/семантика**: расширенные roles/labels/hints с бэка, контроль порядка фокуса, флаги снижения анимации.
- **Аналитика и логирование**: hook‑точки в engine/render/actions, структурированные события, sampling, приватные фильтры.
- **Перфоманс**: префетч экранов/экшенов, lazy‑подсказки по картинкам, skeleton‑пресеты, метрики бюджета рендера.
- **Безопасность**: подпись payload, allowlist экшенов, sanitation инпутов, троттлинг шумных действий.
- **Тесты и тулзы**: golden/snapshot наборы, экспорт схемы контракта, CLI‑валидатор, dev‑меню для моков.
- **Наблюдаемость**: трейсинг стадий (fetch/map/render), тайминги в ответах, экспозиция feature‑тоглов.
